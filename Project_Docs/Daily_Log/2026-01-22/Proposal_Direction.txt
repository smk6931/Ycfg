--------------------------------------------------
- 트렌드 수집 시스템 구축 완료 (Nate, Yahoo JP, Reddit 등)
- 데이터베이스(PostgreSQL) 적재 파이프라인 완성
- AI를 활용한 기초적인 마케팅 키워드 추출 기능 구현 (OpenAI API 연동)
- 프론트엔드 대시보드를 통한 실시간 모니터링 구현

2. 최종 목표 (North Star)
--------------------------------------------------
"자동화된 마케팅 콘텐츠 생성 및 배포 플랫폼 (Marketing Automation Platform)"

[벤치마킹 모델]
- 참조 서비스: https://www.mirra.my/ko/calendar
- 핵심 기능:
    1) 트렌드 자동 감지 (완료 단계)
    2) AI 기반 콘텐츠 생성 (숏폼, 카드뉴스, 블로그 글 등)
    3) SNS 자동 업로드/스케줄링 (유튜브, 인스타, 스레드 등)
    4) 성과 분석 및 재반영

3. 의사결정 요청 사항 (A vs B)
--------------------------------------------------
현재 단계에서 프로젝트의 다음 마일스톤을 어떻게 잡을지에 대한 매니저님의 의견을 구합니다.

[옵션 A: '트렌드 분석' 고도화 우선 (Depth First)]
- 내용: 현재 구축된 수집/분석 엔진을 극한으로 고도화
- 세부 계획:
    - 수집된 키워드가 '왜' 떴는지 심층 분석하는 AI 로직 강화
    - 가짜 뉴스/광고성 키워드 필터링 정교화
    - 마케팅 적합도 점수(Scoring) 알고리즘 개발
    - 사용자 맞춤형 알림 서비스 (Slack/Email) 연동
- 장점: 데이터의 '질(Quality)'이 확실한 SaaS 형태의 분석 툴 완성
- 단점: 실제 콘텐츠 생성/배포 기능 구현이 늦어짐

[옵션 B: 전체 파이프라인 프로토타입 완성 우선 (Breadth First)]
- 내용: '수집 -> 생성 -> 배포'까지 이어지는 전체 워크플로우(MVP)를 먼저 빠르게 연결
- 세부 계획:
    - (Step 1) 수집: 현재 수준 유지
    - (Step 2) 생성: 템플릿 기반의 간단한 카드뉴스/텍스트 자동 생성기 개발
    - (Step 3) 배포: 인스타그램/스레드 API 연동하여 실제 자동 업로드 테스트
    - 이후 각 단계를 점진적으로 개선
- 장점: 서비스의 '전체 그림(Big Picture)'을 빠르게 검증 가능 (Mirra.my와 유사 구조 확립)
- 단점: 초기 콘텐츠 퀄리티나 분석의 깊이가 얕을 수 있음

4. 개발팀 의견 (Suggestion)
--------------------------------------------------
- 저희는 최종 목표가 '자동화 플랫폼'인 만큼, [옵션 B] 방향성을 선호합니다.
- 이유:
    1. 트렌드 분석만으로는 실제 비즈니스 가치(마케팅 실행) 창출에 한계가 있음
    2. SNS 플랫폼(유튜브, 인스타) API 연동 이슈를 미리 파악하고 해결하는 것이 리스크 관리에 유리
    3. 전체 사이클이 돌아가는 것을 먼저 보고, 병목 구간을 찾아 개선하는 것이 효율적
