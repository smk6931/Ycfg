================================================================================
Light Project 개요: SQLite + 구글 시트 연동 버전
================================================================================

작성일: 2026-01-20
목적: 서버 없이 로컬 DB(SQLite)를 활용하여 데이터를 축적하고, 구글 시트로 실시간 보고


핵심 변경 사항 (Enterprise vs Light)
================================================================================

[Enterprise (기존)]
- 서버: AWS EC2, PostgreSQL
- 비용: 월 $50~70 (서버비 + API)
- 구조: 복잡 (Docker, Alembic, Redis)
- 난이도: 상 (운영/유지보수 어려움)

[Light (변경)]
- 서버: 내 PC (로컬 실행)
- DB: SQLite (파일형 DB, 설치 불필요, 무료)
- 비용: 0원 (무료 API 활용)
- 구조: 단순 (Python + SQLite 파일 1개)
- 보고: 구글 스프레드시트 자동 연동 (실시간 대시보드 역할)


프로젝트 목표
================================================================================

1. 수집 (Collect)
   - 전 세계: Google Trends RSS (무료)
   - 한국: Naver DataLab API (무료)
   - 영상: YouTube Data API (무료 할당량)
   - 뉴스: Google News RSS (무료)

2. 저장 (Store)
   - 로컬 `trend_light.db` (SQLite) 파일에 영구 저장.
   - 서버가 없어도 데이터가 계속 쌓임.
   - 언제든 쿼리(SQL)로 조회 가능.

3. 연동 (Sync)
   - 파이썬 실행 시 DB의 최신 데이터를 구글 스프레드시트로 전송.
   - "새로고침" 한 번에 최신 트렌드 리포트 확인 가능.


시스템 아키텍처
================================================================================

[인터넷] ----------------> [내 컴퓨터 (Python)] ----------------> [구글 클라우드]
(Google/Naver/Youtube)      (수집기 + SQLite DB)                 (Google Sheets)

1. 수집기(Collector): API에서 데이터 가져옴
2. DB 저장소: `C:\Ycfg\data\trend_light.db` 파일에 저장
3. 동기화(Syncer): DB에서 엑기스만 뽑아 구글 시트로 업로드


왜 SQLite + 구글 시트 조합인가?
================================================================================

1. 비용 0원
   - 서버 비용 없음, DB 라이선스 비용 없음.

2. 데이터 주권
   - 내 컴퓨터에 파일 형태로 데이터가 남음.
   - 엑셀 파일처럼 백업/복사가 쉬움.

3. 강력한 기능
   - 엑셀은 100만 행 넘어가면 느려지지만, SQLite는 거뜬함.
   - SQL 문법으로 복잡한 검색/필터링 가능.

4. 편리한 공유
   - 결과물은 구글 시트로 팀원과 쉽게 공유.
   - 모바일에서도 확인 가능.
