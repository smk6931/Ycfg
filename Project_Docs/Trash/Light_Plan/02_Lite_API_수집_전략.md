================================================================================

작성일: 2026-01-20
목적: 비용이 들지 않고 접근이 쉬운 API 위주로 수집 전략 재구성


1. 전 세계 트렌드: Google Trends RSS
================================================================================

전략
---
- 공식 API 대신 RSS 피드 활용 (가장 안정적이고 무료)
- "일일 급상승 검색어" 데이터를 국가별로 가져옴.

수집 데이터
---
- 키워드 (검색어)
- 트래픽 (조회수 구간, 예: "10K+")
- 관련 뉴스 기사 (제목, 링크)
- 관련 이미지 URL

URL 예시
---
- 한국: https://trends.google.com/trends/trendingsearches/daily/rss?geo=KR
- 미국: https://trends.google.com/trends/trendingsearches/daily/rss?geo=US
- 일본: https://trends.google.com/trends/trendingsearches/daily/rss?geo=JP

구현 포인트
---
- `feedparser` 라이브러리로 XML 파싱.
- 별도 인증 불필요.


2. 한국 실트: Naver DataLab API
================================================================================

전략
---
- 네이버 개발자 센터 "통합 검색어 트렌드" 및 "쇼핑 인사이트" API 활용.
- 한국 시장에 특화된 정확한 트렌드 파악.

수집 데이터
---
- 쇼핑 인사이트: 분야별 클릭량 추이, 성별/연령별 인기도.
- 데이터랩 검색어: 특정 키워드의 검색량 변화 그래프.

필요 사항
---
- Client ID / Secret 발급 (무료, 즉시 발급)
- 일일 허용량: 25,000회 (충분함)

구현 포인트
---
- HTTP Header에 ID/PW 넣어서 요청.
- JSON 응답 파싱.


3. 영상 트렌드: YouTube Data API (공식)
================================================================================

전략
---
- Apify 대신 구글 공식 API 사용.
- 할당량(Quota) 관리가 핵심 (하루 10,000점).

수집 데이터 (Most Popular)
---
- 영상 제목, 채널명
- 조회수, 좋아요 수, 댓글 수
- 썸네일 URL
- 게시일

API 최적화 (할당량 아끼기)
---
- `videos.list` (chart='mostPopular'): 1회 호출에 1점. (매우 저렴)
- `search.list` (키워드 검색): 1회 호출에 100점. (비쌈, 꼭 필요할 때만 사용)
- 전략: 매일 국가별 인기 영상 Top 50만 긁어오면 하루 5점만 씀. (무제한급)

필요 사항
---
- Google Cloud Console 프로젝트 생성 -> API Key 발급.


4. 실시간 뉴스/이슈: 뉴스 크롤링
================================================================================

전략
---
- 실시간 랭킹 뉴스를 보여주는 포털 활용.
- 사설 사이트(Signal.bz)보다 포털(Nate/Zum)이 안정적.

대상 사이트
---
1. 네이트 랭킹 뉴스 (https://news.nate.com/rank/interest)
   - 사람들이 가장 많이 본 뉴스 1~50위.
2. 줌 인터넷 이슈 검색어 (https://issue.zum.com/)
   - 현재 뜨고 있는 검색어.

구현 포인트
---
- `requests` + `BeautifulSoup` 사용.
- 복잡한 로그인/동적 로딩 없이 HTML만 긁으면 됨.


종합 수집 스케줄 (Local PC 기준)
================================================================================

[실행 방식]
- 파이썬 스크립트(`main.py`)를 실행할 때마다 전체 1회 수집.
- 또는 윈도우 "작업 스케줄러"에 등록해서 하루 3번 자동 실행.

[프로세스]
1. Google Trends (KR, US, JP) 긁기 -> 5초 소요
2. YouTube 인기 영상 (KR) 긁기 -> 2초 소요
3. Naver 데이터랩 확인 -> 1초 소요
4. DB 저장 (SQLite) -> 0.1초 소요
5. 구글 시트 전송 -> 3초 소요
=> 총 15초 내에 전 세계 트렌드 파악 완료.
