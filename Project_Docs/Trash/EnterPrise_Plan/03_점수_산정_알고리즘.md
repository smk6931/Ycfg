================================================================================
핫이슈 점수 산정 알고리즘 설계
================================================================================

작성일: 2026-01-20
목적: 수집된 데이터를 기반으로 핫이슈를 정량적으로 평가하는 알고리즘 정의


점수 산정 개요
================================================================================

핫이슈 점수 = 인기도(40%) + 상승세(30%) + 최근성(20%) + 다출처(10%)

총점: 100점 만점
목표: 객관적이고 재현 가능한 점수 산정 시스템 구축


1. 인기도 점수 (Popularity Score) - 40점
================================================================================

정의
---
해당 이슈가 얼마나 많은 사람들의 관심을 받고 있는지를 측정

측정 지표
---
1. 조회수 (Views)
   - YouTube 동영상 조회수
   - 뉴스 클릭 수 (추정)

2. 참여도 (Engagement)
   - 좋아요 (Likes)
   - 댓글 (Comments)
   - 공유/리포스트 (Shares/Reposts)

3. 언급량 (Mentions)
   - Google Trends 검색량
   - Instagram 해시태그 게시물 수
   - News 기사 수

계산 방법
---
1단계: 각 지표별 원시 점수 수집
   views_score = log10(total_views + 1)
   engagement_score = log10(likes + comments * 2 + shares * 3 + 1)
   mentions_score = log10(mention_count + 1)

2단계: 정규화 (Min-Max Scaling)
   normalized_score = (score - min_score) / (max_score - min_score) * 10

3단계: 가중 평균
   popularity = (
     normalized_views * 0.4 +
     normalized_engagement * 0.4 +
     normalized_mentions * 0.2
   ) * 4

최종 점수: 0~40점


2. 상승세 점수 (Momentum Score) - 30점
================================================================================

정의
---
시간에 따른 관심도 증가율을 측정하여 급상승 트렌드 포착

측정 지표
---
1. 검색량 증가율
   - 현재 시간 대비 24시간 전
   - 현재 시간 대비 1주일 전

2. 게시물 증가율
   - 최근 6시간 게시물 수 vs 이전 6시간
   - 최근 1일 게시물 수 vs 이전 1일

3. 참여도 증가율
   - 시간당 평균 참여도 변화

계산 방법
---
1단계: 증가율 계산
   growth_rate_24h = (current - prev_24h) / (prev_24h + 1) * 100
   growth_rate_7d = (current - prev_7d) / (prev_7d + 1) * 100
   post_growth = (recent_posts - old_posts) / (old_posts + 1) * 100

2단계: 가속도 계산 (급상승 감지)
   acceleration = growth_rate_24h - growth_rate_7d

3단계: 점수 산정
   if acceleration > 200:
     momentum = 30
   elif acceleration > 100:
     momentum = 25
   elif acceleration > 50:
     momentum = 20
   elif growth_rate_24h > 50:
     momentum = 15
   elif growth_rate_24h > 0:
     momentum = 10
   else:
     momentum = 0

최종 점수: 0~30점


3. 최근성 점수 (Recency Score) - 20점
================================================================================

정의
---
이슈의 신선도를 측정하여 오래된 트렌드 배제

측정 지표
---
1. 최초 발견 시각
2. 최근 게시물 시각
3. 게시 빈도

계산 방법
---
1단계: 시간 차이 계산
   hours_since_first = (now - first_discovered).total_hours()
   hours_since_recent = (now - most_recent_post).total_hours()

2단계: 감쇠 함수 적용
   if hours_since_recent < 3:
     recency = 20
   elif hours_since_recent < 12:
     recency = 18
   elif hours_since_recent < 24:
     recency = 15
   elif hours_since_recent < 48:
     recency = 10
   elif hours_since_recent < 72:
     recency = 5
   else:
     recency = 0

3단계: 게시 빈도 보너스
   if posts_last_6h > 10:
     recency += 2
   if posts_last_12h > 20:
     recency += 3

최종 점수: 0~20점 (최대 25점 가능)


4. 다출처 점수 (Multi-Source Score) - 10점
================================================================================

정의
---
여러 플랫폼과 매체에서 동시에 다루어지는 이슈일수록 높은 점수

측정 지표
---
1. 플랫폼 다양성
   - Google Trends: 1점
   - YouTube: 1점
   - Instagram: 1점
   - News: 1점
   - Threads: 1점

2. 매체 다양성
   - 뉴스 소스 수 (서로 다른 언론사)
   - YouTube 채널 수

계산 방법
---
1단계: 플랫폼 개수 계산
   platform_count = unique_platforms.length
   platform_score = platform_count * 2

2단계: 매체 다양성 계산
   if unique_news_sources >= 5:
     source_diversity = 3
   elif unique_news_sources >= 3:
     source_diversity = 2
   else:
     source_diversity = 1

3단계: 최종 점수
   multi_source = min(platform_score + source_diversity, 10)

최종 점수: 0~10점


통합 점수 산정 프로세스
================================================================================

1. 원시 데이터 수집
   {
     "keyword": "트렌드키워드",
     "country": "KR",
     "data_points": [
       {
         "source": "youtube",
         "views": 1000000,
         "likes": 50000,
         "published_at": "2026-01-20T08:00:00"
       },
       {
         "source": "google_trends",
         "search_volume": 85,
         "growth_24h": 150
       }
     ]
   }

2. 지표별 점수 계산
   popularity_score = calculate_popularity(data_points)
   momentum_score = calculate_momentum(data_points)
   recency_score = calculate_recency(data_points)
   multi_source_score = calculate_multi_source(data_points)

3. 최종 점수 합산
   total_score = (
     popularity_score +
     momentum_score +
     recency_score +
     multi_source_score
   )

4. 랭킹 생성
   - 국가별 정렬: total_score 내림차순
   - Top 20 추출
   - 급상승 Top 10 추출 (momentum_score 기준)


국가별 가중치 조정 (옵션)
================================================================================

설명
---
국가별 인구수와 인터넷 사용자 수 차이를 보정

가중치 테이블
---
US (미국):     1.0 (기준)
KR (한국):     0.3
JP (일본):     0.5
TW (대만):     0.2
ID (인도네시아): 0.6

적용 방법
---
adjusted_score = raw_score * country_weight

주의: 초기에는 가중치 없이 시작, 데이터 축적 후 조정


점수 검증 및 보정
================================================================================

1. 이상치 탐지
---
- 점수가 지나치게 높거나 낮은 경우 검증
- 스팸/봇 활동 의심 데이터 필터링

2. 수동 검증
---
- 상위 10개 이슈는 매일 수동 확인
- 부적절한 콘텐츠 필터링

3. 기계학습 개선 (향후)
---
- 사용자 피드백 수집
- 클릭률(CTR) 데이터로 알고리즘 미세조정


구현 예시 (Python 의사코드)
================================================================================

def calculate_hot_score(keyword_data):
    # 1. 데이터 집계
    total_views = sum([d['views'] for d in keyword_data if 'views' in d])
    total_engagement = sum([
        d.get('likes', 0) + 
        d.get('comments', 0) * 2 + 
        d.get('shares', 0) * 3 
        for d in keyword_data
    ])
    
    # 2. 인기도 점수
    popularity = calculate_popularity(total_views, total_engagement)
    
    # 3. 상승세 점수
    momentum = calculate_momentum(keyword_data)
    
    # 4. 최근성 점수
    recency = calculate_recency(keyword_data)
    
    # 5. 다출처 점수
    multi_source = calculate_multi_source(keyword_data)
    
    # 6. 최종 점수
    return {
        'total': popularity + momentum + recency + multi_source,
        'breakdown': {
            'popularity': popularity,
            'momentum': momentum,
            'recency': recency,
            'multi_source': multi_source
        }
    }
