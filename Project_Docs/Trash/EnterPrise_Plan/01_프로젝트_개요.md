================================================================================
트렌드 키워드 자동 수집 및 분석 시스템 - 프로젝트 개요
================================================================================

작성일: 2026-01-20
목적: n8n 워크플로우를 Python 기반 자동화 시스템으로 전환


프로젝트 목표
================================================================================

1. 수집(Collect)
   - 한국, 일본, 대만, 미국, 인도네시아 5개국의 실시간 트렌드 이슈 수집
   - 사용자가 지정한 특정 키워드 관련 자료 수집
   - 수집 데이터를 구조화된 DB(Google Sheets 또는 Airtable)에 자동 저장

2. 선별(Filter)
   - 수집된 데이터에 대한 핫이슈 점수 산정
   - 다양한 지표를 활용한 랭킹 알고리즘 구현
   - 국가별 Top 20 + 급상승 Top 10 자동 추출

3. 발행(Publish)
   - 상위 이슈 기반으로 Threads 게시물(하루 5~10개) 자동 생성
   - 옵션 A: 승인 후 업로드
   - 옵션 B: 완전 자동 업로드
   - (선택) 인스타그램 카드뉴스 자동 제작 및 업로드


핵심 요구사항
================================================================================

1. 데이터 수집 소스
   - 뉴스/웹: 국가별 주요 매체, RSS, 뉴스 API, 합법 크롤링
   - YouTube: API 기반 국가별 인기/조회수 수집
   - Instagram: 해시태그 기반 게시물 수집
   - Threads: 트렌딩 게시물 수집

2. 점수 산정 기준
   - 반응/인기: 조회수, 기사 수, 언급량 등
   - 상승세: 직전 수집 대비 증가율
   - 최근성: 업로드/발행 시각
   - 다출처: 여러 매체/채널 동시 등장 여부

3. 자동화 주기
   - 매일 자동 실행
   - 실시간 데이터 업데이트


기술 스택 (예상)
================================================================================

1. 백엔드 프레임워크
   - FastAPI: REST API 서버 구축
   - Python 3.11+: 메인 개발 언어

2. 데이터베이스
   - PostgreSQL: 메인 데이터 저장소
   - Redis: 캐싱 및 임시 데이터 저장
   - Google Sheets API: 최종 리포트 저장 (선택)

3. 데이터 수집 API
   - Google Trends API (Apify 유료)
   - YouTube Data API v3 (무료)
   - Instagram Scraper (Apify 유료)
   - Google News RSS (무료)
   - Threads API (Meta 공식 또는 사설)

4. 자동화 및 스케줄링
   - APScheduler: Python 기반 작업 스케줄러
   - Celery (선택): 분산 작업 큐

5. AI/LLM
   - OpenAI GPT-4: 게시글 자동 생성
   - LangChain: AI 워크플로우 구성

6. 배포 환경
   - Docker: 컨테이너화
   - AWS EC2 또는 Oracle Cloud: 서버 호스팅
