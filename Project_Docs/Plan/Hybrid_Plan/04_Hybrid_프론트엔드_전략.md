================================================================================
Hybrid 프론트엔드 및 플랫폼 확장 전략
================================================================================

작성일: 2026-01-20
목적: DB에 축적된 데이터를 시각화하고, 사용자 편의성을 제공하기 위한 React 기반 UI 및 확장 전략 수립


1. 프론트엔드 기술 스택
================================================================================

Core
---
- Framework: React (Vite 이용)
- Language: TypeScript (안정적인 데이터 관리)
- State Mgt: TanStack Query (서버 데이터 동기화 최적화)

Styling (Premium Design)
---
- CSS: TailwindCSS (빠르고 유연한 스타일링)
- UI Library: Shadcn/UI (세련되고 모던한 컴포넌트)
- Icons: Lucide React

Charts
---
- Recharts 또는 Chart.js: 트렌드 추이 그래프 시각화


2. 주요 UI/UX 기능 기획
================================================================================

A. 트렌드 대시보드 (Main)
--------------------------------------------------
- "오늘의 급상승 랭킹"을 카드 형태로 나열.
- 국가별(KR/US/JP) 탭 전환 기능.
- 키워드 클릭 시 상세 분석 페이지로 이동.
- 반응형 디자인 (모바일에서도 확인 가능).

B. 키워드 상세 분석 (Detail)
--------------------------------------------------
- 검색량/언급량 추이 그래프 (Line Chart).
- 관련 뉴스 기사 및 유튜브 영상 리스트.
- AI 요약: "이 키워드가 왜 떴나요?" (GPT-4 기반 3줄 요약).

C. 콘텐츠 팩토리 (Content Factory)
--------------------------------------------------
- 수집된 트렌드를 기반으로 생성된 '카드뉴스'나 '블로그 글' 미리보기.
- 에디터 기능: AI가 쓴 글을 사용자가 직접 수정/컨펌.
- [발행] 버튼: 클릭 한 번으로 연동된 플랫폼에 업로드.

D. 설정 및 연동 (Settings)
--------------------------------------------------
- 수집 키워드 설정 (관심사 등록).
- 플랫폼 계정 연동 (Instagram, Threads, Naver Blog 등).


3. 플랫폼 연동 및 확장 전략
================================================================================

목표
---
단순히 "보는" 도구가 아니라 "해결하는" 도구로 확장.

A. 멀티 플랫폼 계정 연동
--------------------------------------------------
- OAuth 2.0 기반 인증 구현.
- 사용자가 자신의 인스타/스레드 계정을 로그인 한 번으로 연결.
- API 토큰 관리 및 자동 갱신 (Refresh Token).

B. 원소스 멀티유즈 (OSMU) 자동화
--------------------------------------------------
- 트렌드 하나를 수집하면 -> 
  1) 인스타용 카드뉴스
  2) 스레드용 텍스트
  3) 블로그용 긴 글
  3가지 포맷으로 자동 변환하여 동시 발행 지원.

C. SaaS(서비스형 소프트웨어)화
--------------------------------------------------
- 초기: 로컬호스트(localhost)에서 나만 쓰는 도구.
- 중기: 사내 서버에 배포하여 팀원과 공유.
- 말기: 회원가입을 받고 구독료를 받는 대중 서비스.

4. 개발 착수 순서
================================================================================

1단계: 백엔드 API 완성 (FastAPI)
   - /trends/rank API
   - /trends/{id} API

2단계: React 프로젝트 세팅
   - `npm create vite@latest`
   - 기본 레이아웃 잡기

3단계: 데이터 연동
   - FastAPI에서 데이터 받아와서 화면에 뿌리기

4단계: 시각화 및 고도화
   - 그래프 추가, UI 디자인 폴리싱
