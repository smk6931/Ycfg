Hybrid Project 개요: PostgreSQL + Vector + AI Ready + Web Service
================================================================================

작성일: 2026-01-20
수정일: 2026-01-20 (프론트엔드 확장 전략 추가)
목적: 무료/유료 API 비교 검증, AI 벡터 DB 구축, 그리고 최종적으로 사용자 친화적 웹 서비스로의 확장.


핵심 전략 (Hybrid & Scalable)
================================================================================

1. 데이터베이스 강화 (PostgreSQL + pgvector)
   - 트렌드 키워드를 **벡터화(Embedding)**하여 저장.
   - 단순 검색을 넘어 "맥락"을 이해하는 AI 검색 기반 마련.

2. API 전면전 (무료 vs 유료)
   - [무료] vs [유료] 소스 동시 운영 및 가성비 검증.
   - 최적의 데이터 파이프라인 구축.

3. 웹 서비스 확장 (Service Vision) [New]
   - 백엔드(Data) 위에 **React 기반의 프론트엔드(UI)** 구축.
   - 개발자 도구(CLI)에서 일반 사용자도 쓰는 **SaaS 형태**로 진화.
   - 타 플랫폼(인스타, 스레드, 블로그 등) 계정 연동 및 자동 발행 허브 역할 수행.


시스템 아키텍처 (Full Stack)
================================================================================

[수집 Layer] ----> [처리 Layer] ----> [DB Layer] ----> [API Layer] <----> [Client Layer]
(API/Crawling)     (Python/AI)       (PostgreSQL)      (FastAPI)          (React App)

1. 수집/처리: 데이터 확보 및 벡터화, 점수 산정.
2. DB: AI 임베딩이 포함된 관계형 데이터 저장소.
3. API: 프론트엔드와 데이터를 주고받는 REST API 서버.
4. Client: 
   - 트렌드 대시보드 (시각화)
   - 콘텐츠 에디터 (AI 생성물 수정)
   - 계정 연동 설정 (OAuth)


단계별 로드맵
================================================================================

Phase 1: 데이터 파이프라인 구축 (현재 단계)
   - 로컬 Docker 환경의 PostgreSQL + pgvector 세팅.
   - 무료/유료 API 수집기 개발 및 DB 적재.

Phase 2: AI 지능화
   - 수집 데이터 임베딩 변환 및 저장.
   - 랭킹 알고리즘 고도화.

Phase 3: React 프론트엔드 개발
   - Vite + React로 대시보드 구축.
   - DB에 쌓인 트렌드를 그래프와 카드로 시각화.
   - "키워드 랭킹"을 한눈에 확인.

Phase 4: 서비스 고도화 (SaaS)
   - 사용자 로그인/회원가입 기능.
   - "내 계정 연동" (Instagram/Threads/Blog 연결).
   - 원클릭 자동 발행 및 예약 시스템 구현.


기대 효과
================================================================================
- 단순 자동화 툴이 아닌, **"트렌드 분석 및 마케팅 자동화 플랫폼"**으로 성장.
- 축적된 벡터 데이터를 기반으로 독보적인 트렌드 인사이트 제공.
- 추후 구독형 서비스로 비즈니스 모델 확장 가능.
